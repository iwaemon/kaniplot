## Per-Feature Details

### PNG output
- **Role:** Physicist creating presentation slides
- **What they tried:** Basic plot, labels, custom range, data file, replot, default terminal
- **Discoverability:** Clear from README
- **Setup:** `cargo build` works
- **Functionality:** All 6 tests PASS. Valid 1250x937 PNG at 150 DPI.
- **Friction points:** No way to control DPI/size (documented as scope-out)
- **Doc suggestions:** Add complete `set terminal png` + `set output` example

### PDF output
- **Role:** Researcher writing LaTeX paper
- **What they tried:** Basic plot, multiple series, data file, script file
- **Discoverability:** Clear from README
- **Setup:** Works
- **Functionality:** All 4 tests PASS. Valid PDF 1.7 documents.
- **Friction points:** None
- **Doc suggestions:** Document page dimensions, font embedding

### LaTeX math + PNG/PDF
- **Role:** Quantum physicist with math-heavy labels
- **What they tried:** Superscripts, Greek letters, fractions, integrals, \sin/\cos
- **Functionality:**
  - PASS: Superscripts, subscripts, Greek letters, basic formulas
  - FAIL: `\sin`, `\cos` not recognized (not in symbol table — these are LaTeX operators, not symbols)
  - FAIL: `\frac` appears as literal text in some contexts (shell escaping issue with printf)
  - FAIL: `|` character confuses parser in `$|\psi|^2$`
- **Note:** These are LaTeX parser limitations, not PNG/PDF conversion issues. The SVG→PNG/PDF pipeline works correctly.

## Issues Found

1. **(LaTeX parser, not this PR)** `\sin`, `\cos`, `\tan` etc. are not recognized as LaTeX commands. They should render as upright "sin", "cos", etc.
2. **(LaTeX parser, not this PR)** `|` character in math mode may interfere with parsing
3. **(Minor)** No way to customize PNG DPI or dimensions (documented as future scope)

## Suggestions

1. Add `\sin`, `\cos`, `\tan` etc. as LaTeX operator commands (future improvement)
2. Add complete examples combining `set terminal` + `set output` in README
3. Document PNG/PDF limitations (fixed DPI, no size control)
