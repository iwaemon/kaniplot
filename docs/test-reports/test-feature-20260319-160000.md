## Per-Feature Details

### Basic math rendering
- **Role:** Scientist creating publication-quality plots
- **What they tried:** `set title "$E = mc^2$"` via pipe
- **Discoverability:** Clear from README
- **Setup:** `cargo build` works
- **Functionality:** PASS — SVG output contains `<tspan>` with Latin Modern Math, correct superscript
- **Friction points:** None
- **Doc suggestions:** None

### Greek letters
- **Role:** Physicist plotting wave functions
- **What they tried:** `set xlabel "$\alpha$"` via echo pipe
- **Discoverability:** Clear from README
- **Setup:** Works
- **Functionality:** PASS when using `printf` or script file. FAIL when using `echo` due to shell escaping (`\a` = BEL, `\b` = backspace, etc.)
- **Friction points:** Shell `echo` interprets backslash sequences before kaniplot receives them
- **Doc suggestions:** Add warning about shell escaping; recommend `printf` or script files

### Font embedding
- **Role:** User checking SVG portability

### Empty math
- **What they tried:** `set title "$$"`
- **Functionality:** PARTIAL — renders correctly (no visible math), but unnecessarily embeds ~1MB font data
- **Impact:** Minor — only affects edge case of literal `$$` in text

## Issues Found

1. **Shell escaping (documentation)** — `echo` interprets `\a`, `\b`, `\f`, `\n`, `\t` as control characters. LaTeX commands like `\alpha`, `\beta`, `\frac`, `\nabla`, `\theta` are affected. Workaround: use `printf` with `\\` or script files. **This is not a kaniplot bug.**

2. **Empty `$` font embedding (minor)** — `$$` triggers font embedding unnecessarily. `model_has_math()` checks for `$` character presence, not for actual math content. Low priority.

## Suggestions

1. **Add shell escaping warning to README** — High impact. Most users will try `echo` first.
2. **Fix `model_has_math()` to check for non-empty math** — Low priority. Edge case optimization.
